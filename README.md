## 📝 개발 방식

`main` branch에서 각자 이름으로 된 branch 생성 후 개발

이후 각자 branch에 PR 올리기!! `main` branch에 merge하지 않도록 주의!

<br>

## 🚩 branch 컨벤션

- 기능 개발 : `feature/{이름}/{기능명}`
- 버그 픽스 : `fix/{이름}/{기능명}`

<br>

## 🚩 커밋 컨벤션

- `feat` : 새로운 기능 추가
- `fix` : 버그 수정
- `docs` : 문서 수정
- `build` : 빌드 관련 파일 수정
- `style` : 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우
- `refactor` : 코드 리팩토링
- `chore` : 그 외 자잘한 수정에 대한 커밋


<br>

### ✅ PR은 템플릿에 따라 작성!
